# Split Embedded Incident

Use this workflow when a Company, Product, or ProductLine page contains detailed Incident content.

## Skills

- `crw-incident-splitter`
- `crw-relationship-mapper`
- `crw-source-auditor`
- `crw-style-lint`
- `crw-ai-policy-finalizer`
- `crw-edit-packager`

## Steps

1. Fetch or paste the parent page wikitext.
2. Identify paragraphs or sections that describe a discrete incident.
3. Confirm the incident is not already covered by another page.
4. Draft the new Incident page.
5. Replace the parent detail with a short neutral summary and link.
6. Validate Cargo relationships.
7. Audit sources.
8. Package both edits.

## Output

- Proposed Incident title.
- New Incident page draft.
- Parent page patch.
- Cargo field rationale.
- Edit summaries for both pages.
